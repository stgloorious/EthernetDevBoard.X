Macro Definition Documentation

◆ ARP_LINK_BROADCAST

#define ARP_LINK_BROADCAST   true

Replies are sent using a link-level broadcast and not unicast.

Enabling this option leads to a quicker conflict detection, but does also increase traffic.

Warning
NOT RECOMMENDED for general use, but other specification may choose to use broadcast for ARP replies if appropriate.
See also
RFC 5227, Section 2.6, Page 14f.

Announces the usage of an ip address (Gratuitous ARP)

This function has to be called repeatetly until it returns ERROR_CODE_SUCCESSFUL According to RFC 5227, a gratuitous ARP announce MUST be performed after the probing of an address

Returns

◆ arp_probe()

static error_t arp_probe ( ipv4_address_t  ipTarget,
linkState_t  link 
)
static

Sends probes to check for address conflicts.

Sends out ARP_PROBE_NUM probes, with an initial random delay (ARP_PROBE_WAIT) and a random delay between ARP_PROBE_MIN and ARP_PROBE_MAX after each probe. It will check each time if there is a valid ARP entry (to detect if there was a reply).

Note
This function has to be called repeatedly until it returns an error code different than ERROR_ARP_WAITING .
Parameters
ipTargetThe IPv4 address being probed for
linkCurrent link status of the Ethernet connection so the state machine can be reset if it disconnects
Returns
ERROR_ARP_MAXIMUM_NUMBER_OF_PROBES_REACHED if there was no answer
ERROR_ARP_WAITING if probing process is not done yet
ERROR_ARP_IPv4_ADDRESS_CONFLICT if there was an ARP message with the requested IP address
